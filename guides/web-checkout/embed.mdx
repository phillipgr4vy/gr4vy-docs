The final step to is to add [Gr4vy Embed][embed] to your checkout experience.
**Gr4vy Embed** handles the discovery of available payment methods,
processing of the payment, and creating a transaction.

## Install Gr4vy Embed

There are a few ways to install Gr4vy Embed, either as a React component, Node
library, or straight off your Gr4vy instance's CDN.

```bash
npm install @gr4vy/embed-react --save
```

:::info Using the CDN

When using the CDN the latest version of the library is always pulled straight
from the Gr4vy server for every request. Although this may be easier to
implement, the load load times of the Gr4vy UI might be negatively impacted.

Embed, it's parameters, events, and all the different ways in which it can be
integrated into your UI.

## Initialize the UI

With Gr4vy Embed installed it's now possible to initialize the Gr4vy UI. Gr4vy
expect the ID of your Gr4vy instance (`gr4vyId `), the amount and currency of
the transaction, the country to process this transaction in, and the client
token you generated in the previous step.

<Tabs
  groupId="language"
  defaultValue="react"
  values={[
    { label: 'React', value: 'react', },
    { label: 'Node', value: 'node', },
    { label: 'CDN', value: 'cdn', },
  ]}
>
  <TabItem value="react">

```js
import Gr4vyEmbed from (`@gr4vy/embed-react)
// or: const { default: Gr4vyEmbed } = require(`@gr4vy/embed-react`)

<form action='/' id='payment-form'>
  <Gr4vyEmbed
    gr4vyId='acme'
    form='#payment-form'
    amount={1299}
    currency='USD'
    country='US'
    token={token}
  />
  <input type="submit" />
</form>
```

  </TabItem>

  <TabItem value="node">

```html
<form action="/checkout" id="payment-form">
  <div class="container" />
  <input type="submit" />
</form>

<script>
  import { setup } from (`@gr4vy/embed`)
  // or: const { setup } = require(`@gr4vy/embed`);

  setup({
    gr4vyId: "acme",
    element: ".container",
    form: "#payment-form",
    amount: 1299,
    currency: "USD",
    country: "US"
    token: token,
  });
</script>
```

:::info Attaching the UI

Gr4vy Embed needs to be attached to a HTML element. In this case we attached the
UI to a `<div>` with the class `container`. The UI can be attached to any
element using any [`querySelector`][queryselector]-compatible query.

:::

  </TabItem>

   <TabItem value="cdn">

```html
<form action="/checkout" id="payment-form">
  <div class="container" />
  <input type="submit" />
</form>

<script src="path/to/gr4vy-embed.js"></script>
<script>

    gr4vyId: "acme",
    element: ".container",
    form: "#payment-form",
    amount: 1299,
    currency: "USD",
    country: "US"
    token: token,
  });
</script>
```

will heavily depend on the enabled payment services in your account.

### Setup parameters

| Parameter   | Description | Type        |
| ----------- | ----------- | ----------- |
| gr4vyId     | **Required.** The identifier for your Gr4vy Instance. | string |
| element     | **Required.** The HTML element to attach Gr4vy Embed. | string, HTMLElement or Element |
| form        | **Optional.** The HTML from to attach Gr4vy Embed.  When this form is submitted, the Gr4vy Embed will initialize the payment transaction. | string, HTMLElement or Element |
| amount      | **Required.** The amount for the transaction. | number |
| currency    | **Required.** The currency for the transaction. | string |
| country     | **Required.** The country of the buyer. | string |
| token       | **Required.** The JWT client token generated in the previous step. | string |
| intent      | **Optional.** The intent of the transaction. Defaults to `authorize`. | `authorize`, `capture` or `approve` |
| store       | **Optional.** Whether to store the payment details (where possible) for future use. Defaults to `ask`. | `ask` or boolean |
| debug       | **Optional.** Whether to output debug messages. Defaults to `false`. | boolean |
| buyerId     | **Optional.** The identifier of the Buyer to associate payment methods to. | string |
| buyerExternalIdentifier | **Optional.** Your identifier of the Buyer to associate payment methods to. | string |
| externalIdentifier | **Optional.** An external identifer for the transaction. | string |
| environment | **Optional.** The environment for the transaction. Defaults to `production`. | `development`, `staging` or `production` |
| theme       | **Optional.** Used to change the styling of Gr4vy Embed. | ThemeOptions See [Theming](../embed/theme) |
| onEvent     | **Optional.** A callback function that's used to subscribe to events. | (eventName: string, data: any) => void |

### Embed Events

| EventName   | Description |
| ----------- | ----------- |
| formUpdate         | Event emitted when the form has been populated. |
| transactionCreated | Event emitted when the transaction has been created. |
| apiError           | Event emitted when an error occurred in an api request. |


## Catch transaction ID

The Gr4vy Embed UI will handle the capture of any payment details, and then
creates a transaction. Once the transaction has been completed can either emit
the transaction ID as an event, or submit any form it was attached to with the
transaction ID embedded in the form.

<Tabs
  groupId="language"
  defaultValue="react"
  values={[
    { label: 'React', value: 'react', },
    { label: 'Node', value: 'node', },
    { label: 'CDN', value: 'cdn', },
  ]}
>
  <TabItem value="react">

```js
const Gr4vyEmbed = require(`@gr4vy/embed-react`)
// or: import Gr4vyEmbed from (`@gr4vy/embed-react)

<form action='/' id='payment-form'>
  <Gr4vyEmbed
    gr4vyId='acme'
    form='#payment-form'
    amount={1299}
    currency='USD'
    country='US'
    token={token}
    onEvent={
      (eventName, data) => {
        if (eventName === 'transactionCreated') {
          console.log(data.id)
        }
      }
    }
  />
  <input type="submit" />
</form>
```

  </TabItem>

  <TabItem value="node">

```html
<form action="/checkout" id="payment-form">

<script>
```

:::info `form` vs `onEvent`

In this example Gr4vy Embed will both log the transaction ID when a transaction
was successfully created. Additionally, it will insert a hidden HTML input into
the form with the name `gr4vy_transaction_id`.

:::
